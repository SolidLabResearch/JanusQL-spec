### Template and build setup

- Renamed the Bikeshed spec from the starter template to `Janus-QL Specification`.
- Updated the spec shortname to `janus-ql-spec`.
- Replaced the original template editor metadata with the current editor information.
- Split the initial content into an included section file at `sections/01-introduction.bs`.
- Added local boilerplate include files so Bikeshed can build the spec in this repository without depending on missing remote boilerplate during local builds.
- Added a local `logo.include` file for the KNoWS branding used by the spec.
- Kept the build output in `dist/index.html`, which is regenerated by `build.sh`.

### Notes for future edits

- `spec.bs` is the Bikeshed entry point.
- New prose should usually go into a file under `sections/` and then be included from `spec.bs`.
- If Bikeshed warnings appear about boilerplate includes, check the local `*.include` files in the repo root first.
- Run `./build.sh` after editing to confirm the spec still generates.
